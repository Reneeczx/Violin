Concepts For Current Module

关键概念

不是完整术语表，而是只给当前模块真正需要的 3 个概念。
Entry Point
程序真正开始执行的入口文件或入口函数。
在这个项目里，`js/app.js` 的 `init()` 就是总入口。
Routing
根据 URL 或 hash 决定当前显示哪个页面。
应用入口不会自己处理页面细节，而是把地址映射注册给路由器。
View Container
先准备好页面容器，再把具体 view 挂进去。
这样做可以让应用壳层负责结构，view 只负责自己的页面内容。
Core Code Reading

核心代码解读

每个模块只讲 1 到 2 段关键代码。这里用“应用入口”做示例。
文件：js/app.js 片段 1 / 2

统一创建 view 容器

const views = {};
['home', 'plan', 'tuner', 'progress', 'theory'].forEach((name) => {
  const div = document.createElement('div');
  div.id = `view-${name}`;
  div.className = 'view';
  main.appendChild(div);
  views[name] = div;
});
它在做什么： 先由应用壳层统一建立所有页面容器。
为什么这样设计： 页面结构由入口层统一管理，后续每个 view 只需要接管自己的 DOM 容器。
继续追读： 看 `HomeView.init(views.home)`，理解容器是如何交给具体模块的。
文件：js/app.js 片段 2 / 2

用 routeMap 把 URL 接到模块

const routeMap = {
  '/': { view: 'home', module: HomeView, navRoute: '/' },
  '/plan': { view: 'plan', module: PlanView, navRoute: '/plan' },
};

Object.entries(routeMap).forEach(([pattern, config]) => {
  router.register(pattern, { ... });
});
它在做什么： 把地址、页面容器和对应模块收口到一个映射表里。
为什么这样设计： 当页面变多时，入口层仍然只维护一张清楚的路由表，而不是散落很多条件判断。
继续追读： 打开 `js/router.js`，看 `register()` 和 `start()` 如何消费这些配置。
